## 1. 个人网站：从0到1的极速搭建与内容运营

### 1.1 技术选型与30分钟上线流程
| 方案 | 技术栈 | 成本 | 推荐指数 | 备注 |
|---|---|---|---|---|
| Hugo+GitHub Pages | Hugo+GitHub Actions | 0元 | ★★★★★ | 静态生成，国内访问快 |
| Hexo+Vercel | Hexo+Vercel CLI | 0元 | ★★★★ | 部署自动化，支持CDN |
| Next.js+GitHub Issues | Next.js+Vercel | 0元 | ★★★ | 动态博客，适合技术展示 |

**极速上线步骤（以Hugo为例）：**
1. 安装Hugo：`brew install hugo`（Mac）或下载Windows二进制包
2. 创建站点：`hugo new site myblog`
3. 添加主题：`git submodule add https://github.com/olOwOlo/hugo-theme-even themes/even`
4. 创建第一篇文章：`hugo new posts/cpp-memory-management.md`
5. 本地预览：`hugo server -D`
6. 部署到GitHub Pages：
   - 创建`username.github.io`仓库
   - 在`config.toml`设置`baseURL = "https://username.github.io"`
   - 推送代码后自动部署

## 2. GitHub开源：从贡献者到项目Owner

### 2.1 首次开源贡献流程
**Step1：寻找适合的项目**
- 搜索标签：`good first issue` + `C++`
- 推荐项目：
  - [fmtlib/fmt](https://github.com/fmtlib/fmt)（格式化库）
  - [catchorg/Catch2](https://github.com/catchorg/Catch2)（测试框架）

**Step2：贡献流程**
```bash
# 1. Fork项目
# 2. 克隆到本地
git clone https://github.com/yourname/project.git
# 3. 创建分支
git checkout -b fix-typo-in-docs
# 4. 修改代码（如修复文档错别字）
# 5. 提交PR
git add .
git commit -m "docs: fix typo in README"
git push origin fix-typo-in-docs
```
