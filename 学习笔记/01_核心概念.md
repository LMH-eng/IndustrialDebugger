## 1. Qt 信号槽机制

### 什么是信号槽？

```
信号 = 事件发生了（按钮被点击、收到数据、连接成功）
槽  = 要执行的函数（响应代码）
connect = 把信号和槽绑在一起
```

### 基本语法

```cpp
connect(发送者, &发送者类::信号, 接收者, &接收者类::槽);
```

### 三种写法

```cpp
// Qt5 推荐写法（编译时检查，安全）
connect(m_connectBtn, &QPushButton::clicked,
        this, &MainWindow::onNetworkConnectClicked);

// Qt4 旧写法（不推荐）
connect(m_connectBtn, SIGNAL(clicked()),
        this, SLOT(onNetworkConnectClicked()));

// Lambda 表达式（简单逻辑用）
connect(m_networkManager, &NetworkManager::connected,
        [this]() { m_statusLabel->setText("网络已连接"); });
```

### 底层原理

1. MOC 编译器扫描 Q_OBJECT 宏
2. 生成 moc_*.cpp 文件
3. 运行时通过元对象系统调用槽函数

### 面试题

**Q: 信号槽的参数有什么限制？**
- 信号的参数 >= 槽的参数
- 信号多余的参数会被忽略

## 2. Qt 内存管理

### 核心规则

**父对象销毁时，子对象自动销毁**

```cpp
// 项目中的例子
m_networkManager = new NetworkManager(this);  // this 是父对象
// 不需要 delete，MainWindow 销毁时会自动删除

MainWindow::~MainWindow()
{
    // 空！子对象自动销毁
}
```

### 设置父对象的方式

```cpp
new QPushButton(this);      // 构造函数传入
btn->setParent(this);       // setParent()
layout->addWidget(btn);     // 添加到布局时自动设置
```

### 面试题

**Q: 什么情况下需要手动 delete？**
- 没有父对象的对象
- 使用了智能指针的情况

## 3. 初始化列表 vs 构造函数体内赋值

```cpp
// 初始化列表（推荐）
MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
    , m_sendCount(0)
    , m_recvCount(0)
{
}

// 构造函数体内赋值（不推荐）
MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
{
    m_sendCount = 0;
    m_recvCount = 0;
}
```

**区别：**
- 初始化列表更高效（直接初始化）
- 构造函数体内赋值是先默认构造再赋值
- const 成员和引用成员必须在初始化列表初始化
